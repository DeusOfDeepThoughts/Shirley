# Shirley
## 功能：
1. 通过提示词工程，对 AI 行为做出约束，让 AI 按照使用者要求进行精读论文；

## 使用方法：
1. 下载后直接点击运行；
2. 用法如下：
  * 新用户建议先设置自定义规则；
  * Shirley 在自定义规则中已添加默认规则，若用户不进行规则客制化，则 Shirley 按照默认规则进行；
  * 其中主要关注的是“重点关注”模块，首页 和 自定义规则页都设置了“重点关注”模块，若用户在首页设置了“重点关注”，则 Shirley 将根据首页设置项进行规划，否则按照自定义规则页的默认设置项进行规划；
  * “重点关注”模块支持用户上传以文档、图片等形式记录的关注点，例如用户可以上传 focus.png，其中 focus.png 包含用户记录的关注点如：实验流程、公式推导 etc.；
  * Shirley 要求用户必须上传至少 1 篇待精读的文档；
  * 待上述流程完成后，用户点击生成任务包，此时 Shirley 生成的 任务包.zip 会下载到本地；
  * 对于 ChatGPT 等支持 zip 格式的 AI，通常用户只需直接上传 zip 到 AI；对于 Kimi 等不支持 zip 格式的 AI，需解压后上传文件内全部材料到 AI。
  * 用户可以选择任何喜欢的 AI 对任务包进行分析，生成相应的精读报告；

## 优势：
1. 针对精读论文，用户可以选择想用的任何 LLM；
2.
  * 本地 agent 在本地工具调用上更有优势（如绘图等），但一般都有 token 限额，如果只是精读一篇论文，只是否值得使用有限的 token 去做一件本可以交给在线语言模型就能完成的任务？
  * API 调用方式可以方便用户直接在对话框中完成任务安排，但对于精读量较大的学生群体而言，这样很烧钱；
  * 如果直接将在线语言模型入口接入 Shirley，尽管可以简化操作流程，但存在账号安全风险；
  * Shirley 当前的设计安排，本质上是希望广大学生群体能省钱、省 token，把钱、token 用在刀刃上；对于没有太多选择的群体，也可以通过 Shirley + 任何免费的在线语言模型，享受比较优质的精读论文服务；
3. Shirley 默认配置了诸如：角色设定、低幻觉、重点关注、输出要求 等全面严谨的约束项，用户可以按照自己的偏好进行修改；
